## 1. 写作顺序：12章优先级排序

### 排序原则
1. **逻辑依赖**: 基础概念先行，应用技术后置
2. **难度梯度**: 从易到难，建立写作节奏
3. **素材匹配**: 优先使用素材丰富的章节
4. **读者价值**: 先解决最痛的问题

### 推荐写作顺序

#### **第1批次（第1-2周）: 基础框架**
**顺序理由**: 建立全书核心思维框架，为后续章节提供理论基础

|--------|------|------|----------|----------|------|
| 1 | 第1章 | 先别自责（系统思维） | 5天 | 高 | 全书哲学基础，其他章节的前提 |
| 2 | 第2章 | 目标管理（不可思议清单） | 5天 | 中 | 为第3章"计划"提供目标输入 |
| 3 | 第3章 | 计划三层（年-周-日） | 5天 | 高 | 将目标转化为可执行计划 |

**顺序理由**: 有了目标和计划，解决"如何开始"和"如何专注"

| 4 | 第4章 | 动量启动（战胜拖延） | 5天 | 高 | 解决"知道但不做"的核心痛点 |

#### **第3批次（第5-7周）: 学习系统**
**顺序理由**: 效率系统的核心能力——输入→处理→输出

## 3. 质量标准：确保 STYLE_GUIDE.md 合规性

### 每章质量检查清单

#### **结构检查（必查项）**
- [ ] **开场痛点** (1-2个典型场景)
- [ ] **本章承诺** (明确具体)
- [ ] **核心原理** (简洁解释)
- [ ] **方法/流程** (分步骤、可复用)
- [ ] **例子** (至少1个学生 + 1个工作场景)
- [ ] **工具箱** (清单/模板/设置项)
- [ ] **常见误区** (3-5条)
- [ ] **7天挑战** (每日小动作)

#### **内容质量检查**
- [ ] **字数范围**: 10,000-15,000字
- [ ] **语气检查**: 坚定、务实、友好（无道德评判）
- [ ] **具体性**: 每个建议都有"条件+步骤"
- [ ] **最小版本**: 每个方法都有MVP版本
- [ ] **前后对照**: 例子使用"之前→之后"结构

#### **术语一致性检查**
- [ ] 术语首次出现时标注英文
- [ ] 使用统一术语表：
  - 注意力切换成本（switching cost）
  - 时间盒（timeboxing）
  - 深度工作（deep work）
  - 主动回忆/检索练习（active recall）
  - 间隔重复（spaced repetition）
  - 费曼技巧（Feynman technique）
  - 周复盘（weekly review）
  - 默认动作（default action）
  - 摩擦（friction）

#### **引用规范检查**
- [ ] 无医疗建议（睡眠/焦虑/成瘾仅给一般性建议）
- [ ] 不神化单一工具或习惯
- [ ] 不许诺"立刻改变人生"
- [ ] 引用分三档处理：
  - 原则级（常识，不标注）
  - 研究级（脚注或延伸阅读）
  - 工具级（名称与用途）

#### **格式规范检查**
- [ ] 标题层级：# → ## → ###
- [ ] 列表使用项目符号，步骤用编号
- [ ] 强调用**加粗**，避免全篇高亮
- [ ] 清单使用可复制格式（短句、动词开头）

### 质量保证流程

#### **Chapter-level QA**（每章完成后）
1. **自检**: 作者对照检查清单自查
2. **交叉审查**: 另一章作者审查
3. **STYLE_GUIDE扫描**: 自动化工具检查术语一致性
4. **修正**: 根据反馈修改

#### **Batch-level QA**（每3章完成后）
1. **逻辑连贯性**: 章节间引用一致
2. **进度一致性**: 例子难度逐渐递进
3. **术语统一性**: 跨章节术语表统一

#### **Final QA**（全书完成后）
1. **全文一致性**: 全书术语、风格、例子统一
2. **完整性**: 所有章节、附录、模板完整
3. **可执行性**: 所有建议都可直接应用
4. **可读性**: 目标读者能轻松理解

### 素材使用策略

#### **分层使用法**
1. **核心素材**（5-8个/章）: 深度挖掘，详细引用
2. **支撑素材**（10-15个/章）: 补充观点，交叉验证
3. **背景素材**（剩余）: 浏览了解，保持上下文

#### **素材处理流程**
```
原始 .srt → 阅读标记 → 提取关键观点 → 归类到章节 → 融入正文
```

#### **素材引用规范**
- ✅ **允许**: 转述核心观点、提炼方法论、改编案例
- ❌ **禁止**: 逐字大段引用、直接翻译字幕
- 🎯 **目标**: 把素材转化为"作者的声音"，而非"编译"

#### **素材管理工具**
建议创建 `material/chapter_mapping.md` 记录每个素材的用途：
```markdown
| 素材编号 | 标题关键词 | 对应章节 | 拟用内容 | 优先级 |
|----------|-----------|----------|----------|--------|
| 003 | Gaining Momentum | Ch4 | 四步启动法 | 核心 |
| 006 | Note-Taking Methods | Ch8 | 五种笔记法对比 | 核心 |
```

## 6. 风险预案：可能遇到的问题及应对方案

### 风险识别与应对矩阵

| 风险类型 | 风险描述 | 发生概率 | 影响程度 | 应对方案 |
|----------|----------|----------|----------|----------|
| **素材风险** | 某章节素材不足 | 中 | 中 | 1. 提前2周检查素材覆盖<br>2. 从其他章节借用相关素材<br>3. 补充外部研究资料 |
| **进度风险** | 某章超期 | 高 | 中 | 1. 每周预留0.5天缓冲<br>2. 降低字数要求（9,000字亦可）<br>3. 调整后续章节顺序 |
| **质量风险** | 不符合 STYLE_GUIDE | 中 | 高 | 1. 每章完成后立即检查<br>2. 建立自动化术语检查工具<br>3. 双周交叉审查 |
| **连贯性风险** | 章节间逻辑断裂 | 低 | 高 | 1. 每3章进行一次连贯性审查<br>2. 维护"术语表"和"概念索引"<br>3. 保持"系统思维"主线 |
| **健康风险** | 作者倦怠/生病 | 低 | 高 | 1. 每周休息1天<br>2. 准备备用作者<br>3. 延长工期1-2周 |
| **工具风险** | Git冲突/文件丢失 | 低 | 中 | 1. 每日 Git commit<br>2. 使用分支隔离<br>3. 定期推送到远程 |
| **版权风险** | 素材使用不当 | 低 | 高 | 1. 转述而非翻译<br>2. 限制单素材引用长度<br>3. 法律顾问审查 |

### 具体应对预案

#### **预案A: 素材不足**
**触发条件**: 某章节核心素材 < 5个

**应对步骤**:
1. **重新评估**: 该素材是否真的必要？能否用常识/研究替代？
2. **素材复用**: 从其他章节借用相关素材（如拖延方法可多章共用）
3. **外部补充**: 
   - 引用经典书籍（《原子习惯》《深度工作》等）
   - 参考学术论文（学习科学、心理学）
   - 使用公开案例研究
4. **降级方案**: 减少该章例子数量，但保证核心方法完整

#### **预案B: 进度落后**
**触发条件**: 连续2周未完成目标

**应对步骤**:
1. **诊断分析**: 落后原因是什么？（素材难、写作慢、审查严）
2. **快速调整**:
   - 降低字数要求（从12,000字降到9,000字）
   - 简化例子（从2个降到1个）
   - 延长工期（总体延长1-2周）
3. **质量不妥协**: 宁可延期，不降低质量标准
4. **重新规划**: 更新后续章节的时间表

#### **预案C: 质量不达标**
**触发条件**: STYLE_GUIDE 检查清单通过率 < 80%

**应对步骤**:
1. **立即停笔**: 不继续写新章节，先修复当前章节
2. **对照检查**: 逐条检查 STYLE_GUIDE 要求
3. **强化审查**: 增加交叉审查次数
4. **建立模板**: 为每章创建标准模板，避免遗漏要素
5. **工具辅助**: 开发自动化检查工具（术语一致性、结构完整性）

#### **预案D: 连贯性问题**
**触发条件**: 审查发现章节间矛盾或重复

**应对步骤**:
1. **建立索引**: 维护"概念索引"（每个概念在哪章出现）
2. **统一术语**: 更新所有章节的术语表
3. **去重处理**: 删除重复内容，保留最完整版本
4. **交叉引用**: 在相关章节间添加引用（"详见第X章"）
5. **主线检查**: 确保"系统思维"主线贯穿全书

#### **预案E: 健康问题**
**触发条件**: 作者连续3天无法工作

**应对步骤**:
1. **立即暂停**: 健康优先，项目暂停
2. **评估影响**: 预计恢复时间，更新时间表
3. **备用方案**: 
   - 启用备用作者（需要1周交接）
   - 延长工期1-2周
4. **降低压力**: 恢复后降低每日产量（从3,000字降到2,000字）

## 7. Git 工作流：分支策略与提交规范

### 分支策略

#### **主分支结构**
```
master (主分支，稳定版本)
  ├─ draft/chapter-01 (第1章草稿分支)
  ├─ draft/chapter-02 (第2章草稿分支)
  ├─ ...
  ├─ draft/chapter-12 (第12章草稿分支)
  └─ review/batch-1 (第1批次审查分支)
  └─ review/batch-2 (第2批次审查分支)
  └─ ...
```

#### **分支命名规范**
- `draft/chapter-XX`: 单章节草稿分支
- `review/batch-X`: 批次审查分支
- `fix/issue-XX`: 问题修复分支
- `final/polish`: 最终打磨分支

#### **工作流程**

**1. 单章节写作流程**
```bash
# 1. 从 master 创建草稿分支
git checkout master
git checkout -b draft/chapter-01

# 2. 在草稿分支写作
# ... 每日提交 ...
git add book/chapter-01.md
git commit -m "feat(ch01): 完成'系统思维'初稿（8,000字）"

# 3. 章节完成时创建 PR
git push origin draft/chapter-01
# 在 GitHub/GitLab 创建 Pull Request

# 4. 审查通过后合并到 master
git checkout master
git merge draft/chapter-01
git branch -d draft/chapter-01
```

**2. 批次审查流程**
```bash
# 1. 创建批次审查分支
git checkout master
git checkout -b review/batch-1

# 2. 合并该批次的所有章节
git merge draft/chapter-01
git merge draft/chapter-02
git merge draft/chapter-03

# 3. 进行批次审查（连贯性、术语统一性）
# ... 修改 ...
git add book/
git commit -m "review(batch-1): 统一术语、修复交叉引用"

# 4. 审查通过后合并回 master
git checkout master
git merge review/batch-1
git branch -d review/batch-1
```

### 提交规范（Conventional Commits）

#### **提交消息格式**
```
<type>(<scope>): <subject>

<body>

<footer>
```

#### **类型（type）**
- `feat`: 新增章节或主要功能
- `fix`: 修复错误或问题
- `style`: 格式调整（不影响内容）
- `docs`: 文档更新
- `refactor`: 内容重构（不改功能）
- `review`: 审查修改
- `test`: 测试相关（本项目中少用）

#### **作用域（scope）**
- `ch01` ~ `ch12`: 具体章节
- `batch-1` ~ `batch-5`: 批次审查
- `global`: 全局性修改
- `appendix`: 附录

#### **示例**

**新增章节初稿**
```bash
git commit -m "feat(ch01): 完成'系统思维'初稿

- 痛点场景2个（拖延、自我怀疑）
- 核心原理（效率三层、系统思维）
- 方法论（环境设计、默认动作）
- 例子2个（学生、工作者）
- 工具箱1个（触发器清单）

字数: 8,234字
素材使用: 003, 018, 056, 084, 116
"
```

**修复审查问题**
```bash
git commit -m "fix(ch04): 修复 STYLE_GUIDE 审查问题

- 补充'7天挑战'部分（之前遗漏）
- 增加第2个例子（工作场景）
- 统一术语：friction -> 摩擦
- 降低道德评判语气

审查来源: STYLE_GUIDE checklist
"
```

**批次审查**
```bash
git commit -m "review(batch-1): 第1批次术语统一

主要修改：
- 统一'timeboxing'为'时间盒（timeboxing）'
- 第2章补充对第1章的引用
- 去重：删除第3章与第1章重复的内容
- 概念索引：创建 CONCEPT_INDEX.md

影响范围: ch01, ch02, ch03
"
```

### PR 创建时机
